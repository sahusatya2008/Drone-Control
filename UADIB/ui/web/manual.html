1. What UADIB Does (Simple Overview)

UADIB is a universal drone interface builder. When you connect a drone endpoint, it attempts to detect protocol, profile the drone, map capabilities, auto-load plugins, and generate a control dashboard. You can operate through UI buttons or API endpoints.

2. Prerequisites (Before You Start)

1. Python virtual environment support installed.
2. Run command from project root: /Users/drhacker/Documents/drone/UADIB.
3. Make sure no firewall blocks local port used by UADIB.
4. Keep token ready. Default token is changeme unless overridden by UADIB_AUTH_TOKEN.
5. For real drones, confirm legal flight area and hardware arm-safety checklist.

5. Simulation Demo: Full Procedure

Goal: verify complete stack without real drone risk.
Step A: keep default simulator source mavlink://udp:127.0.0.1:14550.
Step B: click Run Demo Flow.
Step C: UADIB executes connect -> takeoff -> telemetry sample -> land.
Step D: verify dashboard JSON appears and facilities list is populated.
Expected result: no API errors, telemetry shows altitude/speed/battery/gps data.

6. Real Drone Connect: Full Procedure

1. Connect drone transport physically/network first.
2. Enter endpoint in Real Drone Connect.
3. Example endpoint forms:
  - mavlink://udp:192.168.1.50:14550
  - serial:///dev/ttyUSB0
  - ros2://topic/drone
  - sdk://drone.local
4. Click Connect Real Drone.
5. Confirm profile + capabilities + plugins are returned.
6. Click Refresh Dashboard to fetch latest capability layout.

7. Flight Controls: What Every Button Does

Takeoff: sends target takeoff altitude.
Land: sends landing command, speed resets to 0.
Hover: immediate hold behavior by setting speed 0.
RTL: return-to-launch request.
Set Speed: updates target speed in meters/second.
Set Yaw: updates heading (0-360 degrees).
Best practice: connect first, then takeoff, then speed/yaw changes.

8. Movement + Camera Controls

Move Delta: relative move using delta_lat, delta_lon, delta_alt.
Start Camera: turns camera stream state ON.
Stop Camera: turns camera stream state OFF.
Check telemetry after each action to verify state updates.

9. Telemetry Operations

Start Live Telemetry: subscribes to /ws/telemetry.
Stop Live Telemetry: closes WebSocket cleanly.
Fetch Snapshot: one-time call to /drone/telemetry.
Typical telemetry fields: altitude, speed, battery, gps, armed, camera_streaming, geofence/failsafe flags.

10. Mission Builder

Enter 3 waypoint lines in format lat,lon,alt.
Click Build + Simulate Mission.
Backend returns optimized waypoint order and simulation values:
- distance_m
- eta_s

11. Safety Geofence

Set min/max latitude and longitude bounds.
Click Apply Geofence.
Telemetry will include inside/outside fence checks.
Use conservative boundaries for first test flights.

12. Common Errors and Fixes

{"detail":"Not Found"}: wrong path. Use root UI / or API docs /docs.
401 Unauthorized: token mismatch. Update Access token to match UADIB_AUTH_TOKEN.
Address already in use: rerun master_run.sh; it frees port or auto-selects next.
WebSocket disconnect traceback: fixed in latest build; restart server if using old process.
No drone connected: connect with Simulation or Real endpoint before sending controls.

14. Operator Safety Checklist (Always Do This)

1. Confirm battery health before arm/takeoff.
2. Verify GPS lock if navigation is needed.
3. Keep geofence active for unknown locations.
4. Test hover and land before mission execution.
5. Keep emergency land/RTL ready.
